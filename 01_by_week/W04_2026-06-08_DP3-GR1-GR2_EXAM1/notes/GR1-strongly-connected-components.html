1. Definition

Strongly Connected: Vertices v and w are strongly connected if there's a path v â†‌ w AND a path w â†‌ v.

Strongly Connected Component (SCC): A maximal set of mutually strongly connected vertices.

"Maximal" means you can't add any more vertices and still have everyone strongly connected.

2. The Metagraph â€” Every Directed Graph Is a DAG of Its SCCs

If you collapse each SCC into a single "meta-vertex" and keep the edges between them, you get a DAG.

graph LR
    S1["SCC: A,B,E"] --> S2["SCC: C,D"]
    S1 --> S3["SCC: F"]
    S2 --> S3

Key Property: The metagraph of SCCs is always a DAG.

Why? If there were a cycle between two SCCs S and S', then every vertex in S could reach every vertex in S' and vice versa â€” meaning they should be merged into one bigger SCC. Contradiction.

3. The Goal

We want to find all SCCs of a directed graph in O(n + m) time.

Strategy: Think of the metagraph as a DAG. If we can identify the SCCs one at a time (say, starting from sinks), we can peel them off.

Key insight: If we pick any vertex v in a sink SCC and run Explore(v), we visit exactly the vertices in that SCC (and nothing else â€” because a sink SCC has no outgoing edges in the metagraph).

So the plan is:

  1. Find a vertex in a sink SCC
  2. Explore from it â†’ discover the entire sink SCC
  3. Remove it
  4. Repeat

4. Finding a Sink SCC via the Reverse Graph

Problem: How do we find a vertex guaranteed to be in a sink SCC?

ًں’، Intuition: We can't directly find a sink SCC, but we CAN find a source SCC â€” it's the one containing the vertex with the highest post number after DFS.

And here's the trick: sources in G^R are sinks in G (and vice versa).

G^R is the reverse graph â€” same vertices, all edges flipped:

  • E^R = {(w, v) : (v, w) âˆˆ E}

The SCCs are the same in G and G^R (reversing edges doesn't change strong connectivity). But sources and sinks swap!

The Key Lemma

Lemma: The vertex with the highest post number (from DFS on G^R) lies in a source SCC of G^R, which is a sink SCC of G.

Proof sketch: For SCCs S and S' with an edge S â†’ S' in the metagraph, the maximum post number in S is greater than the max post number in S'. The SCC with the globally highest post number must therefore be a source (nothing has a higher post number pointing into it). In G^R, this source corresponds to a sink in G.

5. The SCC Algorithm

SCC(G):
    Input:  Directed graph G = (V, E)
    Output: ccnum[v] = SCC number for each vertex

    1. Construct G^R (reverse all edges)
    2. Run DFS(G^R) â€” compute post numbers
    3. Order vertices by decreasing post number
    4. Run DFS(G) using this vertex ordering
       â€” each call to Explore from the main loop discovers a new SCC

Runtime: O(n + m)

  • Step 1: O(n + m) to construct reverse graph
  • Step 2: O(n + m) for DFS
  • Step 3: O(n) for ordering (or just process in decreasing order)
  • Step 4: O(n + m) for DFS

Why Step 4 Works

In Step 4, we process vertices in decreasing post-number order from G^R. The first vertex we pick is in a sink SCC of G. Explore visits exactly that SCC (it can't escape â€” no outgoing metagraph edges). We mark it and continue. The next unvisited vertex with the highest remaining post number is in the next sink SCC (after removing the previous one). And so on.

7. The Proof â€” Why This Works

The Core Claim

Claim: For SCCs S and S', if there's an edge from S to S' in the metagraph, then: max post-number in S > max post-number in S' (in the DFS on G^R)

Wait â€” actually the claim is stated for G^R's DFS. Let me state it precisely.

Lemma: In DFS on G^R, the vertex with the highest post number lies in a source SCC of G^R (= sink SCC of G).

Proof:

Consider two SCCs S and S' where S' â†’ S is an edge in G^R's metagraph (meaning S â†’ S' in G's metagraph).

Let u be the first vertex visited (by DFS on G^R) among all vertices in S âˆھ S'.

Case 1: u âˆˆ S. Then from u, DFS can reach all of S (since S is an SCC). DFS will finish all of S before it can possibly finish exploring from u. But S has an edge to S' in G^R's metagraph, so DFS might also reach S'. Either way, post(u) â‰¥ max post in S', and u âˆˆ S, so max post in S > max post in S'.

Case 2: u âˆˆ S'. Then DFS explores all of S' from u. But S' has no edge to S in G^R, so DFS won't reach S from within S'. All of S' finishes before any of S is visited. So max post in S > max post in S'.

In both cases, max post in S > max post in S'. The SCC with the globally highest post number has no SCC with a higher max-post pointing to it â†’ it's a source in G^R â†’ sink in G. âœ…
