1. What Is a Graph?

A graph G = (V, E) consists of:

  • V — a set of vertices (aka nodes)
  • E — a set of edges connecting pairs of vertices
graph LR
    A --- B
    A --- C
    B --- C

Two flavors:

  • Undirected: edges are unordered pairs {u, v}. If A connects to B, then B connects to A.
  • Directed: edges are ordered pairs (u, v). A → B does NOT mean B → A.
graph LR
    A -->|directed| B
    A -->|directed| C
    B -->|directed| C

We assume all graphs are simple — no self-loops (u, u) and no parallel edges.

2. Terminology

⚠️ Common Pitfall: In directed graphs, adjacency has direction. "A is adjacent to B" means there's an edge A → B. It does NOT mean B → A.

3. Graph Representations

Adjacency Matrix

An n × n matrix M where M[i][j] = 1 if edge (i, j) exists, 0 otherwise.

Example (undirected):        Example (directed):
    1  2  3                      1  2  3
1 [ 0  1  1 ]               1 [ 0  1  1 ]
2 [ 1  0  1 ]               2 [ 0  0  1 ]
3 [ 1  1  0 ]               3 [ 0  1  0 ]
  • Space: O(n²)
  • Check if edge exists: O(1) ✅
  • List all neighbors: O(n) ❌
  • Undirected → symmetric matrix

Adjacency List

Each vertex stores a list of its neighbors.

Example (directed):
1 → [2, 3]
2 → [3]
3 → [2]
  • Space: O(n + m) where m = |E|
  • Check if edge exists: O(degree) ❌
  • List all neighbors: O(degree) ✅

💡 Intuition: For sparse graphs (m ≪ n²), adjacency lists win. Most real-world graphs are sparse. This course uses adjacency lists unless stated otherwise.

Adjacency Matrix Adjacency List
Space O(n²) O(n + m)
Edge lookup O(1) O(degree)
Iterate neighbors O(n) O(degree)
Best for Dense graphs Sparse graphs (most cases)

4. BFS: Breadth-First Search

BFS explores the graph level by level, like ripples in a pond. It finds shortest paths in unweighted graphs.

BFS(G, s):
    Input:  G = (V, E), source vertex s
    Output: dist[v] = min edges from s to v
            prev[v] = predecessor on shortest path

    for all v ∈ V:
        dist[v] = ∞
        prev[v] = null

    dist[s] = 0
    Q = queue containing s

    while Q is not empty:
        v = dequeue(Q)
        for all edges (v, w) ∈ E:
            if dist[w] = ∞:          // not yet visited
                dist[w] = dist[v] + 1
                prev[w] = v
                enqueue(Q, w)

Runtime: O(n + m) — each vertex and edge visited once.

BFS Example

graph LR
    A((A<br/>d=0)) --> B((B<br/>d=1))
    A --> C((C<br/>d=1))
    B --> D((D<br/>d=2))
    C --> D
    C --> E((E<br/>d=2))

Starting from A: BFS visits {A} → {B, C} → {D, E}

🎯 Exam Tip: BFS gives shortest paths in unweighted graphs only. For weighted graphs, you need Dijkstra's.

5. Dijkstra's Algorithm

Dijkstra's is BFS for weighted graphs (with positive weights only).

Dijkstra(G, s, w):
    Input:  G = (V, E), source s, weights w(e) > 0 for all e
    Output: dist[v] = shortest weighted path from s to v
            prev[v] = predecessor on shortest path

    for all v ∈ V:
        dist[v] = ∞
        prev[v] = null

    dist[s] = 0
    H = min-heap with all vertices, keyed by dist

    while H is not empty:
        v = extractMin(H)
        for all edges (v, w) ∈ E:
            if dist[v] + w(v,w) < dist[w]:
                dist[w] = dist[v] + w(v,w)
                prev[w] = v
                decreaseKey(H, w)

Runtime: O((|V| + |E|) log |V|) using a binary min-heap.

Key lemmas:

  1. The subpath of any shortest path is itself a shortest path
  2. Triangle inequality: dist(u, v) ≤ dist(u, x) + dist(x, v)

⚠️ Common Pitfall: Dijkstra's does NOT work with negative edge weights. For negative weights, use Bellman-Ford (covered in DP3).

6. DFS: Depth-First Search (Undirected)

DFS explores as deep as possible before backtracking. It's the foundation for SCCs, topological sort, and cycle detection.

DFS for Finding Connected Components

DFS(G):
    Input:  G = (V, E) undirected, adjacency list
    Output: ccnum[v] = connected component number for each v

    cc = 0
    for all v ∈ V:
        visited[v] = false

    for all v ∈ V:
        if not visited[v]:
            cc = cc + 1
            Explore(v)

Explore(z):
    visited[z] = true
    ccnum[z] = cc
    for all (z, w) ∈ E:
        if not visited[w]:
            Explore(w)

Runtime: O(n + m)

💡 Intuition: Each call to Explore from the main DFS loop discovers a new connected component. Everything reachable from z gets the same cc number.

7. DFS for Directed Graphs — Pre/Post Numbering

For directed graphs, we add a clock to track the order of discovery (pre) and completion (post) for each vertex.

DFS(G):
    Input:  G = (V, E) directed, adjacency list
    Output: pre[v], post[v] for each vertex

    clock = 1
    for all v ∈ V:
        visited[v] = false

    for all v ∈ V:
        if not visited[v]:
            Explore(v)

Explore(z):
    pre[z] = clock; clock++
    visited[z] = true
    for all (z, w) ∈ E:
        if not visited[w]:
            Explore(w)
    post[z] = clock; clock++

Runtime: O(n + m)

Worked Example

graph LR
    A --> B
    A --> C
    B --> D
    B --> E
    D --> E
    E --> A

Running DFS starting from A:

Vertex pre post
A 1 10
B 2 9
D 3 6
E 4 5
C 7 8

The pre/post intervals tell us about the tree structure:

  • Containment: If [pre(u), post(u)] contains [pre(v), post(v)], then u is an ancestor of v
  • Disjoint: If intervals don't overlap, neither is an ancestor of the other

8. Edge Types in DFS

When we run DFS on a directed graph, every edge falls into exactly one of four categories:

Edge Type Relationship Post Numbers
Tree edge Parent → child in DFS tree post(z) > post(w)
Back edge Descendant → ancestor post(z) < post(w) ⚠️
Forward edge Ancestor → non-child descendant post(z) > post(w)
Cross edge Between unrelated vertices post(z) > post(w) 
graph TD
    A -->|tree| B
    A -->|tree| C
    B -->|tree| D
    D -->|back| A
    A -->|forward| D
    C -->|cross| D

⚠️ Common Pitfall: Back edges are the ONLY type where post(z) < post(w). For all other edges, post(z) > post(w). This is critical for understanding cycles and topological sort.

The Cycle Theorem

G has a cycle ⟺ its DFS tree has a back edge.

Proof (→): Consider a cycle C. Let v_i be the first vertex of C visited by DFS. All other cycle vertices are reachable from v_i, so they become descendants. The edge v_{i-1} → v_i goes from descendant to ancestor = back edge. ✅

Proof (←): A back edge v → w means v is a descendant of w. So there's a path w ↝ v in the tree, plus the edge v → w. That's a cycle. ✅

9. Topological Sorting

A topological ordering of a DAG is a linear ordering of vertices such that for every edge u → v, u comes before v.

💡 Intuition: Think of it as a valid scheduling order. If task A must happen before task B, then A appears first.

The Algorithm: Run DFS, then sort vertices by decreasing post number.

TopologicalSort(G):
    Input:  DAG G = (V, E)
    Output: vertices in topological order

    1. Run DFS(G) to compute post numbers
    2. Return vertices sorted by decreasing post[v]

Why it works: In a DAG, there are no back edges. So for every edge u → v, post(u) > post(v). Sorting by decreasing post number gives a valid ordering.

Runtime: O(n + m) for DFS + O(n) for sorting = O(n + m)

Example

graph LR
    X["X (post=10)"] --> Y["Y (post=8)"]
    X --> Z["Z (post=6)"]
    Y --> Z
    Z --> U["U (post=4)"]
    Z --> W["W (post=2)"]

Topological order: X → Y → Z → U → W (sorted by decreasing post number)

Alternative algorithm (useful conceptually):

  1. Find a sink vertex (no outgoing edges), output it
  2. Remove it from the graph
  3. Repeat until graph is empty
  4. Reverse the output list

🎯 Exam Tip: Topological sort only works on DAGs. If the graph has a cycle, topological ordering is impossible. (The existence of a back edge proves the cycle.)

💡 The Big Picture: DFS is the Swiss Army knife of graph algorithms. Connected components, cycle detection, topological sorting, SCCs — they all build on DFS. Master DFS and the rest follows.
