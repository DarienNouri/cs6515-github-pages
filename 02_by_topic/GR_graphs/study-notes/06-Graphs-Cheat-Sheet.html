Algorithms at a Glance

Algorithm Input Output Runtime Key Idea
BFS G, source s dist[], prev[] O(n+m) Queue; shortest paths (unweighted)
Dijkstra's G, s, w(e)>0 dist[], prev[] O((V+E)log V) Min-heap BFS; positive weights only
DFS G pre[], post[], ccnum[] O(n+m) Recursion; foundation for everything
Topo Sort DAG ordered vertices O(n+m) DFS, sort by decreasing post number
SCC Directed G SCC labels O(n+m) DFS on G^R, then DFS on G
2-SAT 2-CNF formula assignment or NO O(n+m) Implication graph + SCC
Kruskal's Undirected G, weights MST O(E log V) Sort edges; Union-Find
Prim's Undirected G, weights MST O(E log V) Grow tree; min-heap

Key Lemmas & Theorems

Cycle Detection

G has a cycle ⟺ DFS tree has a back edge

SCC Source Lemma

Vertex with highest post number (in DFS on G^R) is in a source SCC of G^R = sink SCC of G

2-SAT Lemma

f is satisfiable ⟺ ∀i, xᵢ and x̄ᵢ are in different SCCs

Cut Property (MST)

A minimum-weight edge crossing any cut (S, S̄) is in some MST. Proof idea: If MST T doesn't contain e*, add e* (creates cycle), remove another cut edge e'. w(T') ≤ w(T).

Ergodic Theorem (Markov Chains)

Irreducible + aperiodic ⟹ unique stationary distribution π, and Pᵏ → π regardless of start.

Edge Types in DFS (Directed Graphs)

Type Direction post(z) vs post(w)
Tree Parent → child post(z) > post(w)
Back Descendant → ancestor post(z) < post(w) ⚠️
Forward Ancestor → descendant post(z) > post(w)
Cross No ancestor relation post(z) > post(w)

Only back edges have post(z) < post(w). Everything else: post(z) > post(w).

SCC Algorithm — Quick Steps

  1. Build G^R (reverse edges)
  2. DFS(G^R) → get post numbers
  3. Order vertices by decreasing post number
  4. DFS(G) in that order → each Explore = one SCC

2-SAT — Quick Steps

  1. Build implication graph: clause (α ∨ β) → edges ᾱ→β and β̄→α
  2. Run SCC algorithm
  3. If any xᵢ and x̄ᵢ in same SCC → unsatisfiable
  4. Assign: sink SCC = TRUE, its complement (source) = FALSE, repeat

MST — Quick Reminders

  • Kruskal's: Sort edges, add lightest if no cycle (Union-Find)
  • Prim's: Grow one tree, add lightest cut edge (min-heap)
  • Both correct via Cut Property
  • Trees: n-1 edges, connected, acyclic (any 2 ⟹ 3rd)

PageRank — Quick Formula

$$\pi(x) = \sum_{y \in in(x)} \frac{\pi(y)}{|out(y)|}$$

  • Equivalent to πP = π (stationary distribution)
  • Damping: P' = αP + (1-α)·(1/N)·J, with α ≈ 0.85
  • Sink nodes: random jump (α = 0)
  • Compute via power iteration: μ · P'^k → π

⚠️ Common Pitfalls

Pitfall Why It's Wrong
Using Dijkstra's with negative weights Can give wrong shortest paths; use Bellman-Ford
Topological sort on a cyclic graph Impossible — check for back edges first
Confusing 2-SAT implication direction (α ∨ β) gives ᾱ→β, NOT α→β
Assuming MST is unique Multiple MSTs possible if edges have equal weights (total weight is unique)
Assuming webgraph is ergodic It's not — damping parameter is essential
Forgetting Markov chains need stochastic matrix Every row must sum to 1

🎯 Exam Tips

  1. DFS is the Swiss Army knife — SCCs, topo sort, cycle detection all build on it
  2. Cut Property proof — high probability exam question. Know the swap argument.
  3. 2-SAT construction — practice converting clauses to implication edges quickly
  4. SCC order matters — DFS on G^R first, then G. Not the other way around.
  5. Runtimes are all O(n+m) for graph decomposition algorithms (BFS, DFS, SCC, 2-SAT, topo sort). Only MST requires sorting: O(E log V).
  6. PageRank = stationary distribution — know the three equivalent views (citation, random walk, eigenvector)
