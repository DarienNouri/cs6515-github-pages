GR2: 2-Satisfiability (2-SAT)

📚 Reference: DPV Chapter 3.4 (Applications of SCC) This is a beautiful application of SCCs. We reduce a logic problem to a graph problem and solve it in linear time.

1. SAT and CNF — The Setup

Boolean Formulas

We work with boolean variables x₁, x₂, ..., xₙ, each either TRUE (1) or FALSE (0).

For each variable xᵢ, we have two literals: xᵢ and x̄ᵢ (its negation/complement).

Conjunctive Normal Form (CNF)

A formula in CNF is an AND of OR-clauses:

f = (x2) ∧ (x̄3 ∨ x4) ∧ (x3 ∨ x̄5 ∨ x̄1 ∨ x2)

  • Clause: An OR (∨) of literals, e.g., (x₃ ∨ x̄₅ ∨ x̄₁)
  • Formula: An AND (∧) of clauses
  • Satisfying assignment: Values for each variable that make the entire formula TRUE
    • This means at least one literal in every clause must be TRUE

The SAT Problem

Input: A formula f in CNF with n variables and m clauses Output: A satisfying assignment, or "NO" if none exists

2. k-SAT

k-SAT restricts every clause to have at most k literals.

k Complexity
1 Trivial (each clause forces a variable)
2 Polynomial — O(n + m) via SCCs!
3+ NP-complete (no known polynomial algorithm)

The jump from k=2 to k=3 is one of the most important complexity boundaries in CS. We're going to solve 2-SAT.

3. The Implication Graph

Here's the key idea: every 2-SAT clause can be rewritten as two implications.

For a clause (α ∨ β):

  • If α is FALSE, then β MUST be TRUE → implication: ᾱ → β
  • If β is FALSE, then α MUST be TRUE → implication: β̄ → α

Construction

Given a 2-SAT formula with n variables and m clauses:

  • Vertices: 2n vertices — one for each literal xᵢ and x̄ᵢ
  • Edges: 2m directed edges — two implications per clause
Clause Edge 1 Edge 2
(α ∨ β) ᾱ → β β̄ → α

Example

f = (x̄1 ∨ x̄2) ∧ (x2 ∨ x3) ∧ (x̄3 ∨ x̄1)

Clause Implications Edges
(x̄₁ ∨ x̄₂) x₁ → x̄₂, x₂ → x̄₁ x₁ → x̄₂, x₂ → x̄₁
(x₂ ∨ x₃) x̄₂ → x₃, x̄₃ → x₂ x̄₂ → x₃, x̄₃ → x₂
(x̄₃ ∨ x̄₁) x₃ → x̄₁, x₁ → x̄₃ x₃ → x̄₁, x₁ → x̄₃ 
graph LR
    x1["x1"] --> nx2["not-x2"]
    x2["x2"] --> nx1["not-x1"]
    nx2 --> x3["x3"]
    nx3["not-x3"] --> x2
    x3 --> nx1
    x1 --> nx3

💡 Intuition: Paths in this graph are chains of logical implications. If there's a path x₁ ↝ x̄₁, it means "if x₁ is TRUE, then x₁ must be FALSE" — a contradiction (unless x₁ is just FALSE).

4. The Key Lemma

Lemma: A 2-SAT formula f is satisfiable if and only if for all i, xᵢ and x̄ᵢ are in different SCCs of the implication graph.

Equivalently: f is unsatisfiable ⟺ there exists some i where xᵢ and x̄ᵢ are in the same SCC.

Why "same SCC = unsatisfiable"

If xᵢ and x̄ᵢ are in the same SCC, then:

  • There's a path xᵢ ↝ x̄ᵢ (if xᵢ is TRUE, then x̄ᵢ must be TRUE → contradiction)
  • There's a path x̄ᵢ ↝ xᵢ (if x̄ᵢ is TRUE, then xᵢ must be TRUE → same contradiction)
  • Neither assignment works → unsatisfiable

Why "different SCCs = satisfiable"

This is the harder direction. We need to actually construct a satisfying assignment. That's where the algorithm comes in.

5. The 2-SAT Algorithm

2SAT(f):
    Input:  Formula f in 2-CNF with n variables, m clauses
    Output: Satisfying assignment or "unsatisfiable"

    1. Construct the implication graph G (2n vertices, 2m edges)
    2. Run SCC algorithm on G
       → get SCCs in reverse topological order (sink first)
    3. For each i: if xᵢ and x̄ᵢ are in the same SCC, return "unsatisfiable"
    4. For each SCC S (in reverse topological order, sink → source):
         - If literals in S are unassigned:
           - Set all literals in S to TRUE
           - Set all complement literals (in S̄) to FALSE
    5. Return assignment

Runtime: O(n + m) — dominated by the SCC computation.

6. Why the Algorithm Is Correct

The Assignment Strategy

💡 Intuition: Sink SCCs are "safe" to set TRUE because they have no outgoing implications to worry about. Their complements are source SCCs (safe to set FALSE because FALSE can imply anything).

Claim: If S is a sink SCC, then S̄ is a source SCC

Proof:

First, we need a helper fact:

Fact: If there's a path α ↝ β in the implication graph, then there's a path β̄ ↝ ᾱ.

Why? Each edge ᾱ → β in the implication graph came from a clause (α ∨ β), which also produced the edge β̄ → ᾱ. Chaining these along a path gives us the complement path in reverse.

Now, suppose S is a sink SCC. For any α ∈ S:

  • There's no edge from α to any vertex outside S (sink property)
  • By the helper fact, there's no edge from any vertex outside S̄ to ᾱ
  • So S̄ has no incoming edges from outside → S̄ is a source SCC ✅

Why Setting Sink = TRUE Works

When we set a sink SCC S to TRUE:

  • S has no outgoing edges → no implications to propagate
  • S̄ is set to FALSE. S̄ is a source → all edges from S̄ go outward. But FALSE can imply anything (FALSE → anything is vacuously TRUE), so no contradictions arise.

After removing S and S̄, the remaining graph still has valid SCCs, and we repeat.

7. Worked Example

f = (x1 ∨ x2) ∧ (x̄1 ∨ x3) ∧ (x̄2 ∨ x̄3) ∧ (x1 ∨ x3)

Step 1: Build implication graph

Clause Edge 1 Edge 2
(x₁ ∨ x₂) x̄₁ → x₂ x̄₂ → x₁
(x̄₁ ∨ x₃) x₁ → x₃ x̄₃ → x̄₁
(x̄₂ ∨ x̄₃) x₂ → x̄₃ x₃ → x̄₂
(x₁ ∨ x₃) x̄₁ → x₃ x̄₃ → x₁ 
graph LR
    nx1["not-x1"] --> x2["x2"]
    nx2["not-x2"] --> x1["x1"]
    x1 --> x3["x3"]
    nx3["not-x3"] --> nx1
    x2 --> nx3
    x3 --> nx2
    nx1 --> x3
    nx3 --> x1

Step 2: Find SCCs

Let's trace the paths:

  • x̄₁ → x₂ → x̄₃ → x₁ → x₃ → x̄₂ → x₁ (cycle: x₁, x₃, x̄₂, x̄₃ form paths but let's check)
  • x̄₃ → x̄₁ → x₃ (via x̄₁ → x₃) — and x₃ → x̄₂ → x₁ → x₃ (cycle!)

SCC analysis:

  • x₁ → x₃ → x̄₂ → x₁: SCC A = {x₁, x₃, x̄₂}
  • x̄₁ → x₂ → x̄₃ → x̄₁: SCC B = {x̄₁, x₂, x̄₃}

Note: SCC A and SCC B are complements of each other! ✅

Step 3: Check — no xᵢ and x̄ᵢ in the same SCC. ✅ Satisfiable!

Step 4: Assign

SCC A = {x₁, x₃, x̄₂} and SCC B = {x̄₁, x₂, x̄₃}

Which is the sink? A has edges: x₁ → x₃ (internal), x₃ → x̄₂ (internal), x̄₂ → x₁ (internal). Any outgoing? Check all edges from members of A to non-members... If A is the sink:

  • Set SCC A = TRUE: x₁ = T, x₃ = T, x̄₂ = T → x₂ = F
  • Set SCC B = FALSE: x̄₁ = F, x₂ = F, x̄₃ = F → x₁ = T, x₃ = T ✅ (consistent!)

Verify: x₁ = T, x₂ = F, x₃ = T

  • (x₁ ∨ x₂) = (T ∨ F) = T ✅
  • (x̄₁ ∨ x₃) = (F ∨ T) = T ✅
  • (x̄₂ ∨ x̄₃) = (T ∨ F) = T ✅
  • (x₁ ∨ x₃) = (T ∨ T) = T ✅

8. Unit Clause Simplification (Preprocessing)

If a clause has only one literal (a unit clause), we can simplify:

  1. Find unit clause with literal α
  2. Set α = TRUE
  3. Remove all clauses containing α
  4. Remove ᾱ from all remaining clauses (may create new unit clauses or empty clauses)
  5. If any clause becomes empty → unsatisfiable
  6. Repeat

This preprocessing reduces the problem before building the implication graph. After removing all unit clauses, every clause has exactly 2 literals.

9. Summary

Problem Given 2-CNF formula, find satisfying assignment
Key idea Build implication graph, run SCC
Satisfiable iff No xᵢ and x̄ᵢ in same SCC
Assignment Sink SCC → TRUE, its complement (source) → FALSE
Runtime O(n + m)
Contrast 3-SAT is NP-complete!

🎯 Exam Tip: The construction of the implication graph is the critical step. Remember: clause (α ∨ β) gives edges ᾱ → β and β̄ → α. Each clause contributes exactly 2 edges.

⚠️ Common Pitfall: Don't confuse the implication graph edges. For (α ∨ β), the implication is "if NOT α, then β" — so the edge goes from ᾱ to β, not from α.
