GR3: Minimum Spanning Tree

📚 Reference: DPV Chapter 5.1 (Greedy Algorithms — MST) MST is our first greedy algorithm that provably works. The key is the Cut Property.

1. Problem Definition

Given: An undirected, connected graph G = (V, E) with edge weights w(e) Find: A spanning tree T ⊆ E of minimum total weight: min Σ w(e) for e ∈ T

A spanning tree of G is a subset of edges that:

  • Connects all vertices (spans the graph)
  • Has no cycles (it's a tree)
  • Has exactly |V| - 1 edges

2. Tree Properties

These three properties are equivalent for a subgraph with n vertices:

Property Meaning
Connected + n-1 edges → Must be a tree
Connected + acyclic → Must have n-1 edges
n-1 edges + acyclic → Must be connected

Also important:

  • A tree has exactly one path between any pair of vertices
  • Removing any edge disconnects a tree
  • Adding any edge to a tree creates exactly one cycle

💡 Intuition: A tree is the most economical way to keep a graph connected — it uses the minimum number of edges (n-1). An MST does this with minimum total weight.

3. Kruskal's Algorithm

Repeatedly add the lightest edge that doesn't create a cycle.

Kruskal(G, w):
    Input:  Undirected graph G = (V, E) with weights w(e)
    Output: MST edge set X

    1. Sort E by increasing weight
    2. X = ∅
    3. For each edge e = (v, w) in sorted order:
         if adding e to X doesn't create a cycle:
             X = X ∪ {e}
    4. Return X

Cycle Detection with Union-Find

How do we efficiently check "does adding (v, w) create a cycle"?

Use a Union-Find (disjoint set) data structure:

  • Find(v): Returns the component containing v
  • Union(v, w): Merges the components of v and w

Adding edge (v, w) creates a cycle ⟺ Find(v) = Find(w) (already in same component).

Kruskal(G, w):       // with Union-Find
    1. Sort E by increasing weight
    2. X = ∅
    3. Initialize Union-Find with each vertex in its own set
    4. For each edge e = (v, w) in sorted order:
         if Find(v) ≠ Find(w):
             X = X ∪ {e}
             Union(v, w)
    5. Return X

Runtime: O(|E| log |V|)

  • Sorting: O(|E| log |E|) = O(|E| log |V|) since |E| ≤ |V|²
  • Union-Find operations: O(|E| · α(|V|)) ≈ O(|E|) (nearly constant per op)
  • Dominated by sorting

4. The Cut Property — Why Kruskal's Works

What Is a Cut?

Definition: For an undirected graph G = (V, E), a cut is a partition of V into two non-empty sets S and S̄ = V \ S. The cut edges are: cut(S, S̄) = {(v, w) ∈ E : v ∈ S, w ∈ S̄}

The Cut Property Lemma

Lemma (Cut Property): Let X ⊆ E be a subset of some MST T. Let S ⊆ V be such that no edge of X crosses the cut (S, S̄). Let e* be a minimum weight edge crossing the cut. Then X ∪ {e*} ⊆ T' for some MST T'.

In plain English: A minimum-weight edge across any cut is safe to add to the MST.

💡 Intuition: If I give you any tree T that doesn't include e*, I can swap e* in and some other cut edge out, and the weight won't increase (because e* is the lightest across the cut). So there's always an MST containing e*.

Proof of the Cut Property

Setup: Fix G, X, T (an MST containing X), and S.

  • e* is a min-weight edge in cut(S, S̄)
  • No edge of X crosses the cut

Goal: Show X ∪ {e*} ⊆ T' for some MST T'.

Case A: e* ∈ T. Then T' = T. Done. ✅

Case B: e* ∉ T. We construct T'.

  1. Add e* to T → creates a unique cycle C (since T is a tree)
  2. e* = (a, b) crosses the cut (a ∈ S, b ∈ S̄). The cycle C must cross the cut at least one more time (to get back from S̄ to S). Let e' be another edge of C that crosses the cut.
  3. Set T' = T ∪ {e*} \ {e'}

Claim 1: T' is a tree.

  • T' has n-1 edges (added one, removed one)
  • T' is connected: any path in T that used e' can detour through the rest of cycle C (which includes e*)
  • Connected + n-1 edges → tree ✅

Claim 2: T' is an MST.

  • w(e*) ≤ w(e') because e* is minimum weight across the cut and e' also crosses the cut
  • w(T') = w(T) + w(e*) - w(e') ≤ w(T)
  • But T is already minimum weight, so w(T') = w(T) ✅

Claim 3: X ∪ {e} ⊆ T'.*

  • X ⊆ T and X has no edges crossing the cut, so e' ∉ X, meaning X ⊆ T' = T \ {e'} ∪ {e*}
  • e* ∈ T' by construction ✅

🎯 Exam Tip: The proof of the Cut Property is one of the most likely proof questions on the exam. Know the two cases (e* in T or not) and the swap argument.

5. Why Kruskal's Is Correct (via Cut Property)

At each step of Kruskal's, when we add edge e = (v, w):

  • v and w are in different components (otherwise we'd skip)
  • Let S = the component containing v, S̄ = everything else
  • No edge of X (edges added so far) crosses this cut (by definition of components)
  • e is the minimum weight edge crossing this cut (because we process edges in sorted order — all lighter edges either don't cross this cut or were already added)
  • By the Cut Property, adding e is safe ✅

By induction, X is always a subset of some MST. When |X| = n-1, X IS that MST.

6. Prim's Algorithm

Prim's grows a single tree from a starting vertex, always adding the lightest edge connecting the tree to a non-tree vertex.

Prim(G, w):
    Input:  Undirected graph G = (V, E) with weights w(e)
    Output: MST edge set X

    X = ∅
    S = {any starting vertex}
    
    while |X| < |V| - 1:
        e* = minimum weight edge in cut(S, V\S)
        X = X ∪ {e*}
        S = S ∪ {endpoint of e* in V\S}
    
    Return X

Implementation: Use a min-heap keyed by the lightest edge connecting each non-tree vertex to the tree.

Runtime: O(|E| log |V|) with binary heap

Correctness: Direct application of the Cut Property at each step (S = tree vertices so far, no edge of X leaves S since the tree is contained in S).

7. Kruskal's vs Prim's

Kruskal's Prim's
Strategy Global: lightest edge overall Local: lightest edge from current tree
Grows Forest → single tree Single tree from start
Data structure Union-Find Min-heap
Runtime O(|E| log |V|) O(|E| log |V|)
Better for Sparse graphs Dense graphs
Correctness Cut Property (S = component) Cut Property (S = tree so far)

Both are just special cases of the generic "greedy + Cut Property" approach.

⚠️ Common Pitfall: MST is for undirected graphs only. The directed analog (minimum spanning arborescence) is a completely different problem.

⚠️ Common Pitfall: If multiple edges have the same weight, the MST may not be unique — but the total weight is. Any tie-breaking in Kruskal's or Prim's yields a valid MST.

8. Summary

Problem Find minimum-weight spanning tree
Key Lemma Cut Property: min-weight edge across any cut is in some MST
Kruskal's Sort edges, greedily add lightest non-cycle edge. O(|E| log |V|)
Prim's Grow tree from root, add lightest cut edge. O(|E| log |V|)
Proof technique Swap argument: add e*, remove another cut edge, weight doesn't increase
Tree facts n-1 edges, connected, acyclic (any 2 of 3 imply the third)
